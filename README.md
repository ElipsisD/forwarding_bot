# Telegram бот для маршрутных Excel-файлов

Бот принимает `.xlsx`, вытаскивает из таблицы данные по заказам и отправляет водителю одно или несколько изображений, удобных для просмотра с телефона.

## Что попадает в карточку

- номер заказа;
- получатель;
- количество мест;
- сумма и способ оплаты;
- адрес;
- дополнительная информация;
- время работы.

## Установка

```powershell
uv sync
```

Создайте переменную окружения с токеном бота:

```powershell
$env:BOT_TOKEN="123456789:ваш_токен"
uv run python bot.py
```

Также можно скопировать `.env.example` в `.env`: файл загрузится автоматически при запуске. Переменная окружения, заданная в системе, имеет приоритет над значением из `.env`.

## Запуск через Docker Compose

Скопируйте пример настроек:

```powershell
Copy-Item .env.example .env
```

Откройте `.env` и укажите настоящий токен:

```env
BOT_TOKEN=123456789:ваш_токен
```

Запустите бота:

```powershell
docker compose up -d --build
```

Посмотреть логи:

```powershell
docker compose logs -f bot
```

Остановить:

```powershell
docker compose down
```

## Как пользоваться

1. Запустите `uv run python bot.py` или `docker compose up -d --build`.
2. Отправьте боту Excel-файл `.xlsx`.
3. Бот вернет изображения с карточками заказов и запросит стартовый пробег автомобиля.
4. После ввода пробега бот предложит выбрать точку доставки. После выбора будут показаны все данные заказов по этому адресу. Нажмите `ЗАКАЗ ВЫПОЛНЕН`, когда доставка завершена.
5. Бот предложит только оставшиеся точки. Для ошибочного выбора нажмите `Выбрать другую точку`.
6. После последней точки бот запросит финальный пробег и сообщит пройденное расстояние.

Если таблица содержит много строк, бот разобьет результат на короткие страницы — до шести адресов на изображение. Для длинных карточек ограничение по высоте сработает раньше, чтобы текст оставался читаемым на телефоне.

Когда страниц несколько, бот отправит их одним альбомом, чтобы между ними было удобно листать. Telegram ограничивает альбом десятью изображениями; при большем количестве страниц бот отправит несколько альбомов.

Если адрес объединён в Excel или повторяется в нескольких строках, бот соберёт такие заказы в одну карточку с общим адресом.

Состояние текущего маршрутного листа хранится в `data/routes.json`, поэтому статусы доставок не пропадают после перезапуска. Для каждого чата сохраняется последний загруженный маршрут.

## Координаты точек

Чтобы бот определял координаты адресов, укажите ключи API Геосаджеста и API Геокодера Яндекса в `.env`:

```env
YANDEX_SUGGEST_API_KEY=ключ_API_Геосаджеста
YANDEX_GEOCODER_API_KEY=ключ_API_Геокодера
```

Геокодирование выполняется при загрузке маршрутного листа. Сначала API Геосаджеста уточняет адрес и возвращает `uri`, затем API Геокодера получает по нему координаты. Если Геосаджест вернул несколько вариантов для адреса с пометкой `ТК` или «транспортная компания», бот попросит выбрать нужный адрес до запроса стартового пробега. Для остальных адресов используется первая подсказка. В карточке точки отображаются координаты и точность результата. Если геокодер определил только улицу или другой неточный объект, бот помечает такой адрес как недостаточный для точной доставки. Без ключей Яндекса бот работает как раньше, без запросов к сервису.

Для уже выбранной точки доступна кнопка `УТОЧНИТЬ КООРДИНАТЫ`. После нее отправьте ссылку на карточку из Яндекс Навигатора: бот извлечет координаты из ссылки или использует ее `uri` в API Геокодера и сохранит уточненное значение.

## Разработка через Docker Compose

Исходный код подключён в контейнер как volume, поэтому изменения файлов сразу доступны внутри `bot`. Чтобы запущенный Python-процесс подхватил изменения, перезапустите сервис:

```powershell
docker compose restart bot
```
